## 3. Тест-кейсы (5)

### TC-01 — Открытие главной страницы
- **ID:** TC-01
- **Заголовок:** Главная страница доступна и содержит базовые элементы навигации
- **Предусловия:** Django сервер запущен, доступен `http://127.0.0.1:<порт>/`
- **Шаги:**
  1. Открыть `http://127.0.0.1:<порт>/`
  2. Проверить наличие меню, поиска и иконки корзины
- **Ожидаемый результат:** Страница открылась без ошибок, элементы навигации отображаются
- **Скриншот:** `screenshots/CL-01-home.png`

### TC-02 — Открытие карточки товара
- **ID:** TC-02
- **Заголовок:** Карточка товара открывается по ID и показывает данные товара
- **Предусловия:** Django сервер запущен
- **Шаги:**
  1. Открыть `http://127.0.0.1:<порт>/product.html?id=1`
  2. Проверить заполненность: название, цена, изображение, описание
- **Ожидаемый результат:** Данные товара отображаются корректно
- **Скриншот:** `screenshots/CL-05-product-1.png`

### TC-03 — Изменение количества/веса на карточке товара
- **ID:** TC-03
- **Заголовок:** Пересчет итоговой стоимости при изменении количества/веса
- **Предусловия:** Открыта карточка товара `product.html?id=1`
- **Шаги:**
  1. Нажать `+` 2 раза (увеличить количество/вес)
  2. Проверить, что блок “Итого” изменился
  3. Нажать `-` 1 раз
  4. Проверить повторный пересчет “Итого”
- **Ожидаемый результат:** Итоговая сумма пересчитывается в соответствии с выбранным значением
- **Скриншот (место проверки):** `screenshots/CL-06-product-controls.png`

### TC-04 — Оформление заказа (позитивный)
- **ID:** TC-04
- **Заголовок:** Заказ успешно отправляется при заполненных обязательных полях
- **Предусловия:** В корзине есть хотя бы один товар; сервер запущен
- **Шаги:**
  1. Открыть `/cart.html`
  2. Нажать “Оформить заказ”
  3. Заполнить `Имя` и `Телефон` (Email/Адрес/Комментарий по желанию)
  4. Нажать “Отправить”
- **Ожидаемый результат:** Появляется сообщение об успешной отправке; корзина очищается
- **Скриншот (место проверки):** `screenshots/CL-07-cart-page.png`

### TC-05 — Оформление заказа (негативный)
- **ID:** TC-05
- **Заголовок:** Валидация обязательных полей при отправке заказа
- **Предусловия:** Открыта корзина `/cart.html`; сервер запущен
- **Шаги:**
  1. Нажать “Оформить заказ”
  2. Оставить `Имя` и/или `Телефон` пустыми
  3. Нажать “Отправить”
- **Ожидаемый результат:** Отображается ошибка, заказ не отправляется
- **Скриншот (место проверки):** `screenshots/CL-07-cart-page.png`

## 4. Баг-репорты (3)

### BR-01 — Отсутствует favicon `assets/images/icon.jpg`
- **ID:** BR-01
- **Severity:** Low
- **Priority:** Low
- **Заголовок:** Запрос favicon возвращает 404 (файл `icon.jpg` отсутствует)
- **Шаги воспроизведения:**
  1. Открыть `http://127.0.0.1:<порт>/assets/images/icon.jpg`
- **Ожидаемый результат:** Возвращается изображение favicon (HTTP 200)
- **Фактический результат:** HTTP 404, favicon не загружается
- **Вложение:** `screenshots/BR-01-favicon-404.png`

### BR-02 — `product.html` ломается при открытии без `id`
- **ID:** BR-02
- **Severity:** Medium
- **Priority:** Medium
- **Заголовок:** Страница товара не обрабатывает отсутствие параметра `id` и отображает пустой/некорректный контент
- **Шаги воспроизведения:**
  1. Открыть `http://127.0.0.1:<порт>/product.html` (без параметров)
- **Ожидаемый результат:** Показать сообщение “Товар не выбран” или редирект на каталог 
- **Фактический результат:** Часть данных не заполняется/скрипты могут падать, страница выглядит некорректно
- **Вложение:** `screenshots/BR-02-product-no-id.png`

### BR-03 — API принимает некорректный телефон (с буквами)
- **ID:** BR-03
- **Severity:** Medium
- **Priority:** High
- **Заголовок:** `/api/submit-request/` принимает номер телефона с буквами при наличии хотя бы одной цифры
- **Шаги воспроизведения:**
  1. Отправить POST JSON на `http://127.0.0.1:<порт>/api/submit-request/`:
     - `name`: `Test`
     - `phone`: `abc1` (некорректно)
  2. Проверить ответ сервера
- **Ожидаемый результат:** HTTP 400 и сообщение об ошибке валидации телефона
- **Фактический результат:** HTTP 200 и `status: success` (заявка создается)
- **Вложение:** `screenshots/BR-03-invalid-phone-response.png`
